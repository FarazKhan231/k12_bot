# Exploratory Testing Plan for Passage Preparation

## Objective
To identify usability issues, functional defects, and overall user experience concerns on the live URL: https://passagepreparation.com/

## Scope
Focus on key user journeys, including registration, course selection, and payment processes.

## Duration
2 hours

## Approach
1. Navigate through the website as a typical user.
2. Document findings and report any issues.

## Reporting
Use a shared document for real-time updates and findings.

Checks

  1. Verify Homepage Load — Ensure the homepage loads correctly without errors.
  2. Test Registration Process — Check if users can register successfully and receive confirmation.
  3. Course Selection Functionality — Ensure users can browse and select courses without issues.
  4. Payment Gateway Integration — Verify that the payment process works seamlessly and securely.
  5. Responsive Design Check — Ensure the website is usable on various devices (mobile, tablet, desktop).
  6. Error Handling — Test how the system handles invalid inputs or actions.
  7. User Feedback Mechanism — Check if users can provide feedback easily and if it is acknowledged.

Heuristics

  • User Control and Freedom
  • Consistency and Standards
  • Error Prevention
  • Recognition Rather Than Recall
  • Flexibility and Efficiency of Use
  • Aesthetic and Minimalist Design
